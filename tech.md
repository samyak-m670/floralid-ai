# FlorAid: AI Plant Identifier - Technology Stack

This document outlines the primary technologies, libraries, and tools used in the **FlorAid** project and explains the rationale behind choosing them.

## Core Framework & Language

*   **React (v19)**: The foundational JavaScript library for building the user interface. It provides a robust, component-based architecture that makes building interactive, state-driven UI features seamless.
*   **TypeScript**: A strongly typed superset of JavaScript. We use it to ensure type safety across the codebase, improve the developer experience with better autocomplete/Linting, and catch errors at compile time before they reach production.

## Build Tooling & Environment

*   **Vite**: A lightning-fast development server and build tool that replaces Webpack. It provides instant hot module replacement (HMR) during development and highly optimized static assets for production using Rollup under the hood.
*   **Node.js**: The underlying runtime environment required for Vite, script execution, and potentially running the local server.

## Styling & Animations

*   **Tailwind CSS (v4)**: A utility-first CSS framework integrated via the `@tailwindcss/vite` plugin. It allows us to rapidly build custom, responsive designs directly within our JSX without needing to write or maintain separate traditional CSS files.
*   **Lucide React**: A beautiful, consistent, and lightweight icon library used throughout the UI to enhance visual communication.
*   **Motion**: A powerful animation library for React that makes it easy to add fluid, complex animations and transitions (like the layout animations or card reveals) to improve the overall user experience and give the app a premium feel.
*   **clsx & tailwind-merge**: Utility libraries used in tandem to easily construct CSS class strings conditionally and intelligently merge conflicting Tailwind classes, especially useful for reusable UI components.

## AI & Data Processing

*   **Google Gen AI SDK (`@google/genai`)**: The official SDK used to communicate securely and efficiently with Google's Gemini models. This is the core engine behind the app's capability to analyze images, identify plants, and provide detailed botanical responses.
*   **React Markdown**: A component used to safely parse and render the Markdown formatted responses generated by the Gemini API into standard, styled HTML elements on the frontend.

## Backend & Local Data (Optional/Future-proofing)

*   **Better SQLite3 (`better-sqlite3`)**: An extremely fast, synchronous interface for SQLite databases in Node.js. Used for structuring and storing local relational data efficiently.
*   **Express (`express`)**: A minimal backend web framework for Node.js, potentially used alongside Vite to handle custom server-side routing, API endpoints, or database interactions that shouldn't be exposed directly to the frontend client.
*   **Dotenv**: Used to load environment variables (like the `GEMINI_API_KEY`) from a local `.env.local` file into the application context safely without hardcoding sensitive credentials in the source code.
